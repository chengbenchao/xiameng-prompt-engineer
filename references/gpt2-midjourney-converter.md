# 🔄 GPT-Image-2 → Midjourney 转换层 (Converter Guide)

**版本**: v1.0 (2026-04-27)

---

## 📌 概述

本文件指导如何将 GPT-Image-2 社区精选提示词转换为 Midjourney 适配格式，充实夏导引擎标杆库。

GPT-Image-2 和 Midjourney 的核心差异：
- GPT-Image-2 理解自然语言，Midjourney 需要更结构化的参数
- Midjourney 使用 `--ar` 参数控制画幅，GPT-Image-2 在文本中描述
- Midjourney 不擅长生成文字，GPT-Image-2 可以
- Midjourney 的 `--no` 负面参数体系更完善

---

## 🔧 转换规则

### 1. 画幅比例转换
| GPT-Image-2 写法 | Midjourney 转换 |
|-----------------|-----------------|
| "9:16 vertical" | `--ar 9:16` |
| "16:9 horizontal" | `--ar 16:9` |
| "1:1 square" | `--ar 1:1` |
| "4:3" | `--ar 4:3` |

### 2. 参数精确化
| GPT-Image-2 描述 | Midjourney 精确化 |
|-----------------|-------------------|
| "soft light" | 具体化为光位：侧顺光 / 柔光 / 漫射光 |
| "warm tones" | 加入色温：4800K 暖调 |
| "close-up" | 加入焦段：85mm |
| "wide shot" | 加入焦段：24-35mm |

### 3. 画质词追加
GPT-Image-2 案例通常不含 Midjourney 画质词，转换时必须追加：
```
电影感画面，超写实质感，8k 超高清，RAW 原图，影视级调色，IMAX 级渲染
```

### 4. 负面词追加
```
负面提示：低画质，模糊，变形，多余手指，文字水印，动漫风，卡通风，3D渲染
```

### 5. 不兼容场景跳过
以下 GPT-Image-2 特色场景 **不适合** 转换到 Midjourney：
- 需要精确文字渲染的（报纸头版、UI 截图）
- 需要 Image-to-Image 输入的
- 纯依赖 GPT-Image-2 特殊能力的
